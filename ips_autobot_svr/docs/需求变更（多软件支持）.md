# 需求变更（多软件支持）

## 背景

系统在原有客户端 **智灵**（`zhiling`）基础上新增 **翼飞**（`yifei`）。**同一账号**（手机号/用户名 + 密码）登录不变；与客户端相关的数据按 `app` 维度拆分。

## 客户端枚举

| app 值 | 说明 |
|--------|------|
| `zhiling` | 智灵 |
| `yifei` | 翼飞 |

所有与 MAC、会员到期、积分档位、积分变动、充值记录相关的接口必须显式传入 **`app`**（query 或 body，见 [api.md](../api.md)）。

## 数据库变更摘要

### `users` 表

原单列已迁移/删除，改为按客户端后缀区分：

- `mac_addr_zhiling` / `mac_addr_yifei`（各自 **UNIQUE**，可绑定同一物理 MAC 到两个 app；同一 app 内 MAC 仍全局唯一）
- `member_expire_at_zhiling` / `member_expire_at_yifei`（到期日均按日末 `23:59:59`）
- `member_type_zhiling` / `member_type_yifei`
- `score_zhiling` / `score_yifei`
- `30d_score_zhiling` … `365d_score_zhiling` 与 `30d_score_yifei` … `365d_score_yifei`

### 充值表

- 原 `recharge_records` 重命名为 **`recharge_records_zhiling`**
- 新建 **`recharge_records_yifei`**（结构一致）

### `score_record` 表

- 新增 **`client_app`**（`zhiling` | `yifei`），历史记录在迁移脚本中默认标为 `zhiling`。

### 迁移命令

```bash
npm run db:migrate-multi-app
```

**上线前请备份数据库。** 新环境可直接使用当前 `src/db/schema.sql` 执行 `npm run db:init`（已包含多应用结构）。

## API 破坏性变更要点

- 登录 / `GET /api/users/me` / 用户列表项：用户对象含 **`clients.zhiling`** 与 **`clients.yifei`**，不再使用顶层的单一 `mac_addr`、`score`、`member_expire_at` 等字段。
- **MAC**：`GET/PUT /api/users/mac`、`POST /api/users/mac/verify` 必填 **`app`**；`DELETE /api/users/:id/mac` 必填 query **`app`**。
- **充值**：`POST/GET /api/recharge` 必填 **`app`**。
- **积分**：`POST /api/score/change`、`GET /api/score/records` 必填 **`app`**；流水带 **`client_app`**。
- **超管改用户**：`PUT /api/users/:id` 使用 **`clients.zhiling` / `clients.yifei`** 嵌套对象，可分别设置 `member_expire_days`（增加天数）与 **`member_reduce_days`**（减少天数，不低于今日 `23:59:59`）。

完整请求/响应以 **Swagger** 与 **[api.md](../api.md)** 为准。

## 如何调整各客户端「积分策略」（档位数值）

### 方式一：管理接口（推荐）

超级管理员调用 **`PUT /api/users/:id`**，在 body 中设置对应客户端块，例如只改智灵 30 天档位：

```json
{
  "clients": {
    "zhiling": {
      "30d_score": 400,
      "90d_score": 900
    }
  }
}
```

翼飞示例：

```json
{
  "clients": {
    "yifei": {
      "30d_score": 350
    }
  }
}
```

字段名为 JSON 中的 **`30d_score`**（无需带后缀），服务端会写入 `30d_score_zhiling` / `30d_score_yifei` 等列。

### 方式二：SQL（运维）

在确认影响范围后，可直接更新 `users` 表列名，例如：

```sql
UPDATE users SET `30d_score_zhiling` = 400 WHERE id = 1;
UPDATE users SET `365d_score_yifei` = 4000 WHERE phone = '13800138000';
```

**注意：** 列名必须以 `_zhiling` / `_yifei` 结尾；生产环境先备份，避免批量误更新。

## 客户端接入提示

1. 登录后根据当前软件在后续请求中传 **`app=zhiling` 或 `app=yifei`**。
2. 展示「绑定/积分/到期/档位」时，只读 **`clients[当前app]`** 对应字段。
3. Docker 单文件挂载更新 zip 等行为与本次需求无关；仍见 [DEPLOYMENT.md](./DEPLOYMENT.md)。
